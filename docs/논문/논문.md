## 260402 논문 피드백 (정우 참여)

- 학회 다른 곳 알아보기
- 논문에 AI 느낌 남 → 문장 구구절절 보단 간결하고 명확하게
- 논문 양식마다 다르지만 소속 형식 맞게(굳이 학부생, 교수 구분 안해도됨)
- 설계만 기재한다기 보단, 나중에 구현까지 해서 설계→구현 스토리라인으로
- 시스템 구성 및 하드웨어 → 시스템 구성 따로, 하드웨어 따로
- 하드웨어 구조도 좀더 명확하게(레이어 표시 or 구조도 변경)
- 요구사항
	- 설계로 비롯한 요구사항 내용기제(내용 간단히) + 요구사항 표
- 칼만필터 수식 본문 중간 보단 수식을 따로 기재하고 설명
- 소프트웨어 다이어그램 추천 → 흐름 간단히
- 요구 사항 추적성 매트릭스
	- 전 : 그냥 표로만 검증 사항 기재되어있음
	- 후 : 구현후 추적 매트릭스에 따라 검증한 내용 서술
	- + **α : 성능 평가**
- 레퍼런스 최신화(칼만필터 1960)
- 시스템 구현 내용 서술하는 것은 시간(일정, 능력) 사정상 무조건 100% 구현하고 기재하라는 의미는 아님.
- 혹여, 예상하지 못한 상황이라도 생긴다면 그냥 설계만으로 논문 작성 가능하고, 일부 기능만 구현한 내용만 서술 가능.
