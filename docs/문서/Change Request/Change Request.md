# Change Request


요구사항, 아키텍처, 하드웨어 부품 등의 변경이 필요할 경우, 시스템 전체에 미치는 영향을 사전에 분석하고 기준선에 반영하여 프로젝트의 일관성을 유지합니다.


**[변경 통제 프로세스 흐름]**


**Step 1. 변경 요청 (Change Request)**


요구사항 변경이나 새로운 센서 도입 필요성 발생 시 변경 요청서 작성


**Step 2.영향도 분석 (Impact Analysis)**


제안된 변경이 일정, 예산, 타 모듈(인터페이스)에 미치는 파급 효과 분석


**Step 3. 구현 및 추적성 업데이트 (Implementation)**


승인된 변경 사항을 구현하고 관련 설계 문서를 최신화


**[Change Request 명세 양식]**


프로젝트 기준선을 변경하는 주요 사항은 아래 양식을 통해 이력을 남깁니다.


| **항목**                      | **세부 내용**             | **예시**                                                                    |
| --------------------------- | --------------------- | ------------------------------------------------------------------------- |
| **CR ID**                   | 변경 요청 고유 식별 번호        | CR-2604-01                                                                |
| **CR 일자**                   | 해당 사항을 발의한 날짜         | 2026-04-10                                                                |
| **변경 대상**
**(Item)**        | 요구사항, 하드웨어, 소프트웨어 중택일 | 센서 하드웨어 (초음파 센서 수량 변경)                                                    |
| **변경 사유** 
**(Reason)**     | 해당 변경이 필요한 기술적/일정적사유  | 측후방 사각지대 해소를 위해 HC-SR04 센서 2개추가 필요                                        |
| **영향도 분석** 
**(Impact)**    | 일정, 예산, 인터페이스 파급 효과   | - 예산: 1,980원 추가 발생
- 아두이노 핀 할당 변경(GPIO 46, 47 사용)
- 초음파 필터 노드 배열 크기 수정 필요 |
| **관련문서**
**(Traceability)** | 업데이트가 필요한 연관 문서       | 하드웨어 설계도(Layer 3), RTM 문서 업데이트완료                                          |
