**[RACI]**

- **R (Responsible - 실무자)**
	- 실제로 해당 업무를 기획하고 실행하며 땀 흘리는 사람
- **A (Accountable - 책임자)**
	- 최종 의사 결정을 내리고 결과에 무한 책임을 지는 사람 (업무 당 반드시 1명)
- **C (Consulted - 협의자)**
	- 업무 수행 시 기술적 조언이나 피드백을 쌍방향으로 주고받는 전문가
- **I (Informed - 보고 대상)**
	- 진행 상황과 결과를 통보받아야 하는 다음 단계의 이해관계자

RACI


**[체크포인트 및 시너지 포인트 (조율 가이드)]**

- **의사 결정 라인 명확화**
	- 각 업무마다 최종 책임자(A)가 정확히 1명씩 배정되어 있어 의견 충돌 시 누가 결단을 내릴지 명확합니다.
		- ex) HW 구현 중 문제가 생기면 실무는 최정우가 하지만, 최종 설계 변경 승인은 정재웅이 내림
- **HW와 SW의 브릿지 (C - 협의자)**
	- 하드웨어를 구현할 때 SW구현 담당자인 김명진이 C로 들어가 있습니다. 센서 핀을 어디에 꽂을지, 코딩하기 편한 방향이 무엇인지 서로 소통해야 하기 때문입니다. 반대로 SW 구현 시에도 HW구현 담당자인 최정우가 C로 참여하여 모터와 센서의 물리적 한계를 조언합니다.
- **QA의 독립성:** 최정수는 개발(구현)에 직접 참여하지 않으므로, 사용자 및 평가자 입장에서 객관적이고 냉철하게 버그를 잡아낼 수 있는 완벽한 포지션입니다.
